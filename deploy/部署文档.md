## 二、快速部署（一键脚本）

### 2.1 克隆项目

```bash
git clone <仓库地址> /var/www/resource-share
cd /var/www/resource-share
```

### 2.2 运行一键脚本

```bash
sudo bash deploy/install.sh
```

脚本会自动完成以下步骤：

1. 安装系统依赖（Python3、Node.js、Nginx）
2. 创建 Python 虚拟环境并安装后端依赖
3. 初始化 SQLite 数据库
4. 构建 PC 用户前端 (resource-web)
5. 构建管理后台 (resource-admin)
6. 拷贝构建产物到部署目录
7. 配置 Systemd 服务并启动
8. 配置 Nginx 并重启

### 2.3 验证部署

```
前端页面:   http://服务器IP/
管理后台:   http://服务器IP/admin/
API 健康检查: http://服务器IP/api/v1/
API 文档:   http://服务器IP/docs
```

默认管理员账号：`admin` / `admin123`

## 三、手动部署

### 3.1 安装系统依赖

```bash
# Ubuntu
sudo apt update
sudo apt install -y python3 python3-pip python3-venv nodejs npm nginx

# CentOS
sudo yum install -y python3 python3-pip nginx
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs
```

### 3.2 部署后端 API

```bash
cd resource-api

# 创建虚拟环境
python3 -m venv venv
source venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 初始化数据库（建表+预设分类+管理员账户）
python -m scripts.init_db
python -m scripts.seed_data

# 启动测试（确认无报错后 Ctrl+C 停止）
uvicorn app.main:app --host 0.0.0.0 --port 8000
```

### 3.3 构建前端

```bash
# PC 用户前端
cd resource-web
npm install
npm run build
# 构建产物在 resource-web/dist/

# 管理后台
cd ../resource-admin
npm install
npm run build
# 构建产物在 resource-admin/dist/
```

### 3.4 拷贝文件到部署目录

```bash
sudo mkdir -p /var/www/resource-web
sudo mkdir -p /var/www/resource-admin

sudo cp -r resource-web/dist/* /var/www/resource-web/
sudo cp -r resource-admin/dist/* /var/www/resource-admin/
sudo cp -r resource-api /var/www/resource-api/
```

### 3.5 配置 Systemd 服务

```bash
sudo cp deploy/resource-api.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable resource-api
sudo systemctl start resource-api

# 查看状态
sudo systemctl status resource-api
```

### 3.6 配置 Nginx

```bash
sudo cp deploy/nginx.conf /etc/nginx/sites-available/resource-share
sudo ln -sf /etc/nginx/sites-available/resource-share /etc/nginx/sites-enabled/
sudo rm -f /etc/nginx/sites-enabled/default  # 移除默认站点

# 测试配置
sudo nginx -t

# 重启 Nginx
sudo systemctl restart nginx
```

### 3.7 配置 HTTPS（推荐）

```bash
# 安装 Certbot
sudo apt install -y certbot python3-certbot-nginx

# 申请证书（替换为你的域名）
sudo certbot --nginx -d your-domain.com

# 证书会自动续期，验证：
sudo certbot renew --dry-run
```

## 四、部署架构

```
                        ┌─────────────────┐
                        │    Nginx :80     │
                        │  (SSL :443 可选) │
                        └────────┬────────┘
                                 │
                 ┌───────────────┼───────────────┐
                 │               │               │
                 ▼               ▼               ▼
         ┌────────────┐  ┌────────────┐  ┌──────────────┐
         │  PC 前端    │  │ 管理后台    │  │  FastAPI     │
         │ 静态文件    │  │ 静态文件    │  │  :8000       │
         │ /          │  │ /admin/    │  │ /api/        │
         └────────────┘  └────────────┘  └──────┬───────┘
                                                │
                                                ▼
                                         ┌──────────────┐
                                         │   SQLite      │
                                         │  data/app.db  │
                                         └──────────────┘
```

### Nginx 路径映射

| 路径 | 目标 | 说明 |
|------|------|------|
| `/` | `resource-web/dist/` | PC 用户前端 SPA |
| `/admin/` | `resource-admin/dist/` | 管理后台 SPA |
| `/api/` | `http://127.0.0.1:8000` | 后端 API 代理 |
| `/uploads/` | `http://127.0.0.1:8000` | 上传文件服务 |

## 五、配置说明

### 5.1 关键配置项

| 配置 | 位置 | 默认值 | 说明 |
|------|------|--------|------|
| JWT_SECRET | `resource-api/app/config.py` | 硬编码密钥 | **生产环境必须通过环境变量 `SECRET_KEY` 覆盖** |
| 默认分页 | `resource-api/app/config.py` | 12条/页 | `DEFAULT_PAGE_SIZE` |
| access_token有效期 | `resource-api/app/config.py` | 2小时 | `ACCESS_TOKEN_EXPIRE_MINUTES` |
| refresh_token有效期 | `resource-api/app/config.py` | 7天 | `REFRESH_TOKEN_EXPIRE_DAYS` |
| Nginx 上传限制 | `deploy/nginx.conf` | 200MB | `client_max_body_size` |
| API 端口 | `deploy/resource-api.service` | 8000 | uvicorn 监听端口 |

### 5.2 环境变量

```bash
# 设置 JWT 密钥（重要！）
export SECRET_KEY="your-strong-random-secret-key-here"
```

在 systemd 服务文件中添加环境变量：

```ini
[Service]
Environment=SECRET_KEY=your-strong-random-secret-key-here
```

### 5.3 Nginx 大文件上传配置

`nginx.conf` 中已配置 `client_max_body_size 200M;`（位于 server 块顶层），如果需调整上传大小上限，修改 `deploy/nginx.conf` 中的 `client_max_body_size` 值即可。

## 六、运维命令

### 6.1 服务管理

```bash
# API 服务
sudo systemctl status resource-api      # 查看状态
sudo systemctl restart resource-api     # 重启
sudo systemctl stop resource-api        # 停止
sudo systemctl start resource-api       # 启动
sudo journalctl -u resource-api -f      # 查看实时日志

# Nginx
sudo systemctl restart nginx            # 重启
sudo nginx -t                           # 测试配置
```

### 6.2 更新部署

```bash
cd /var/www/resource-share
git pull

# 重新构建前端
cd resource-web && npm install && npm run build && cd ..
cd resource-admin && npm install && npm run build && cd ..

# 拷贝文件
sudo cp -r resource-web/dist/* /var/www/resource-web/
sudo cp -r resource-admin/dist/* /var/www/resource-admin/

# 更新后端依赖并重启
cd resource-api
source venv/bin/activate
pip install -r requirements.txt -q
deactivate
cd ..
sudo systemctl restart resource-api

# 如果 Nginx 配置有变更
sudo cp deploy/nginx.conf /etc/nginx/sites-available/resource-share
sudo nginx -t && sudo systemctl restart nginx
```

### 6.3 数据备份

```bash
# 备份 SQLite 数据库
cp /var/www/resource-api/data/app.db /backup/app.db.$(date +%Y%m%d)

# 备份上传文件
tar -czf /backup/uploads.$(date +%Y%m%d).tar.gz /var/www/resource-api/uploads/
```

## 七、常见问题

### Q1: 启动后访问 502 Bad Gateway

```
# 检查 FastAPI 是否在运行
sudo systemctl status resource-api

# 查看日志
sudo journalctl -u resource-api -n 50

# 常见原因：Python 虚拟环境路径不正确
# 检查 /etc/systemd/system/resource-api.service 中的 ExecStart 路径
```

### Q2: 上传大文件失败

```
# 检查 Nginx 配置
sudo nginx -T | grep client_max_body_size

# 确认 nginx.conf 中的 client_max_body_size 值足够大（默认 200M）
```

### Q3: 前端页面空白 / 路由刷新 404

```
# 确认 nginx.conf 包含 SPA fallback 配置
location / {
    try_files $uri $uri/ /index.html;
}

location /admin {
    try_files $uri $uri/ /admin/index.html;
}
```

### Q4: 管理后台登录后提示"需要管理员权限"

```
# 需要先在数据库中创建一个 admin 角色的用户
# 建议通过 scripts/seed_data.py 创建
cd /var/www/resource-api
source venv/bin/activate
python -m scripts.seed_data
```

### Q5: SQLite 数据库锁定 / 并发写入慢

SQLite 使用 WAL 模式，已支持读写并发。如果高并发场景下出现性能瓶颈：

1. 升级为 PostgreSQL（需要修改 `database.py` 中的连接代码）
2. 或保持现状，资源分享站写入频率远低于读取
