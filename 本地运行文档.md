## 二、快速启动（推荐）

打开 **三个终端**，分别运行以下三个命令：

### 终端 1 — 启动后端 API

```bash
cd resource-api
python -m venv venv

# Windows:
venv\Scripts\activate
# macOS/Linux:
# source venv/bin/activate

pip install -r requirements.txt
python -m scripts.init_db
python -m scripts.seed_data
uvicorn app.main:app --reload --port 8000
```

### 终端 2 — 启动用户前端

```bash
cd resource-web
npm install
npm run dev
```

### 终端 3 — 启动管理后台

```bash
cd resource-admin
npm install
npm run dev
```

启动后访问地址：

| 项目 | 地址 |
|------|------|
| 用户前端 | http://localhost:5173 |
| 管理后台 | http://localhost:5174 |
| API 接口 | http://localhost:8000 |
| API 文档 | http://localhost:8000/docs |

## 三、分步启动

### 3.1 启动后端 API

```bash
cd resource-api

# 1) 创建虚拟环境（仅首次）
python -m venv venv

# 2) 激活虚拟环境
# Windows (PowerShell):
venv\Scripts\Activate.ps1
# Windows (CMD):
# venv\Scripts\activate
# macOS / Linux:
# source venv/bin/activate

# 3) 安装依赖（仅首次，或依赖变更后）
pip install -r requirements.txt

# 4) 初始化数据库（仅首次）
python -m scripts.init_db

# 5) 填充种子数据（仅首次，创建管理员和预设分类）
python -m scripts.seed_data

# 6) 启动开发服务器（热重载）
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

启动后终端输出示例：

```
INFO:     Uvicorn running on http://127.0.0.1:8000
INFO:     Application startup complete.
```

> `--reload` 参数：代码修改后自动重启，开发必备。

### 3.2 启动用户前端

```bash
cd resource-web

# 安装依赖（仅首次）
npm install

# 启动开发服务器
npm run dev
```

启动后终端输出示例：

```
VITE v5.x  ready in 200ms
  ➜  Local:   http://localhost:5173/
  ➜  Network: http://192.168.x.x:5173/
```

> Vite 开发服务器已配置代理（`vite.config.ts`）：
> - `/api/*` → `http://localhost:8000/api/*`
> - `/uploads/*` → `http://localhost:8000/uploads/*`
>
> 前端直接调用 `/api/v1/xxx` 即可，无需手动拼接后端地址。

### 3.3 启动管理后台

```bash
cd resource-admin

# 安装依赖（仅首次）
npm install

# 启动开发服务器
npm run dev
```

启动后终端输出示例：

```
VITE v5.x  ready in 250ms
  ➜  Local:   http://localhost:5174/
  ➜  Network: http://192.168.x.x:5174/
```

> 管理后台的 Vite 代理配置与用户前端相同，也是将 `/api` 代理到 `localhost:8000`。

## 四、验证运行

### 4.1 API 健康检查

浏览器访问 http://localhost:8000/，应返回：

```json
{"code": 0, "message": "资源分享站 API 运行中", "data": null}
```

### 4.2 API 文档

浏览器访问 http://localhost:8000/docs，应看到 Swagger 文档页面，所有接口可在线调试。

### 4.3 用户前端

浏览器访问 http://localhost:5173，应看到：

- 首页：顶部导航栏 + 搜索区 + 热门资源区
- 点击"资源分类"进入分类页
- 点击"上传中心"进入上传页面（需先登录）

### 4.4 管理后台

浏览器访问 http://localhost:5174，使用以下账号登录：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| `admin` | `admin123` | 管理员 |

登录后可见：数据统计面板、资源管理、分类管理、用户管理。

## 五、端口与访问地址

| 项目 | 开发端口 | 启动命令 |
|------|----------|----------|
| 后端 API | `8000` | `uvicorn app.main:app --reload --port 8000` |
| 用户前端 | `5173` | `cd resource-web && npm run dev` |
| 管理后台 | `5174` | `cd resource-admin && npm run dev` |

### 修改端口

**后端**：`--port` 参数指定

```bash
uvicorn app.main:app --reload --port 9000
```

**前端**：在 `vite.config.ts` 中配置

```ts
export default defineConfig({
  server: {
    port: 5173,  // 改为你想用的端口
    proxy: { ... }
  }
})
```

> 管理后台需使用不同端口（如 5174），否则两个前端会冲突。

### 两个前端同时运行的说明

`resource-web` 和 `resource-admin` 是两个独立的 Vite 项目，各自运行在自己的端口上：

```
浏览器标签1: http://localhost:5173   ← 用户端
浏览器标签2: http://localhost:5174   ← 管理后台
```

## 六、开发调试技巧

### 6.1 API 文档调试

FastAPI 自动生成 Swagger 文档：

```
http://localhost:8000/docs      # Swagger UI
http://localhost:8000/redoc     # ReDoc (可选)
```

在 Swagger 页面中，可以：
- 直接调用所有 API 接口
- 点击右上角"Authorize"输入 Token 后测试需登录的接口
- 查看每个接口的请求/响应格式

### 6.2 Token 获取方式

```bash
# 注册
curl -X POST http://localhost:8000/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"test","password":"123456"}'

# 登录（获取 Token）
curl -X POST http://localhost:8000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# 返回示例：
# {"code":0,"data":{"access_token":"eyJ...","refresh_token":"eyJ...","token_type":"bearer"}}
```

### 6.3 直接查看数据库

SQLite 数据库文件位置：`resource-api/data/app.db`

使用 SQLite 客户端查看：

```bash
# 命令行
sqlite3 resource-api/data/app.db
.tables
SELECT * FROM users;
SELECT * FROM categories;
SELECT * FROM resources;

# 或使用 VS Code 插件：SQLite Viewer
```

### 6.4 清除数据重新开始

```bash
# 删除数据库文件
rm resource-api/data/app.db

# 重新初始化
cd resource-api
python -m scripts.init_db
python -m scripts.seed_data
```

### 6.5 Vite HMR 热更新

两个前端项目都支持 Vite HMR（模块热替换）：
- 修改 `.vue` 文件 → 页面实时更新，保留状态
- 修改 `.ts` 文件 → 模块热替换，无需刷新
- 修改后端 `.py` 文件 → `--reload` 自动重启

## 七、常见问题

### Q1: `pip install` 失败

```bash
# 升级 pip
python -m pip install --upgrade pip

# Windows 下如果 bcrypt 编译失败，安装预编译包
pip install --only-binary=:all: -r requirements.txt
```

### Q2: Windows 下运行后端报编码错误

```bash
# 设置 UTF-8 编码
$env:PYTHONIOENCODING="utf-8"

# 或在 PowerShell 启动前执行
[System.Console]::OutputEncoding = [System.Text.Encoding]::UTF8
```

### Q3: 虚拟环境激活失败

```powershell
# PowerShell 执行策略限制，先执行：
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

# 如果 venv 目录不存在，重新创建：
python -m venv venv --clear
```

### Q4: `npm install` 报错

```bash
# 清除缓存重试
npm cache clean --force
npm install

# 或使用淘宝镜像
npm config set registry https://registry.npmmirror.com
npm install
```

### Q5: 前端调用 API 报跨域错误

检查后端是否启动了（`localhost:8000`），以及 Vite 代理是否生效：

```
Network tab 中请求地址应为 http://localhost:5173/api/v1/xxx
而非 http://localhost:8000/api/v1/xxx
```

Vite 代理已配置，只需使用相对路径 `/api/v1/xxx` 即可。

### Q6: 两个前端端口冲突

如果 `npm run dev` 提示端口被占用：

```bash
# 用户前端指定其他端口
cd resource-web
npx vite --port 5173

# 管理后台指定其他端口
cd resource-admin
npx vite --port 5174
```

### Q7: 数据库表不存在

```bash
# 执行初始化脚本
cd resource-api
python -m scripts.init_db
python -m scripts.seed_data
```

### Q8: `scripts` 模块找不到

确保在 `resource-api` 目录下执行命令，且虚拟环境已激活。`init_db.py` 和 `seed_data.py` 中已通过 `sys.path.insert` 处理了模块路径。
